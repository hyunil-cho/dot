# 프로젝트 계획서 (WBS 및 요구사항 명세)

본 문서는 `PRD.md`를 기반으로 작성된 프로젝트 WBS 및 기능 요구사항 명세서 초안입니다.

## 1. 프로젝트 개요
- **프로젝트명**: Dot (가칭) - 감성 음성 응답 시스템
- **목표**: 사용자가 사랑하는 사람의 목소리와 성향을 AI에 학습시켜, 실제 통화와 유사한 경험을 제공함.
- **핵심 가치**: 고인의 재현이 아닌, 제한된 템플릿 기반의 감성적 연결 경험 제공.

## 2. WBS (Work Breakdown Structure)``

### Phase 1: 기획 및 설계 (Planning & Design)
- [ ] **요구사항 상세화**: PRD 기반 기능 명세 확정
- [ ] **UI/UX 설계**:
    - 전화 통화 경험을 극대화하는 UX 시나리오 작성 (비동기 처리 시각화 등)
    - 주요 화면 와이어프레임 및 디자인 (로그인, 전화번호부, 통화 화면 등)
- [ ] **시스템 설계**:
    - DB 스키마 설계 (User, Persona, VoiceData, CallLog 등)
    - API 명세서 작성 (Swagger/OpenAPI)
    - AWS 인프라 아키텍처 설계 (보안 및 네트워크)
- [ ] **보안 정책 수립**: 개인정보 암호화 및 오남용 방지 가이드라인 수립

### Phase 2: 백엔드 개발 (API Server - SpringBoot)
- [ ] **환경 설정**: SpringBoot 프로젝트 세팅 및 DB 연동
- [ ] **회원 관리 모듈**:
    - 회원가입 (이메일/비밀번호), 로그인(JWT), 탈퇴
- [ ] **대상(Persona) 관리 모듈**:
    - 대상 프로필 등록/수정/삭제/조회
    - 성향 정보 관리
- [ ] **데이터 관리 모듈**:
    - 음성 파일 업로드/삭제 (S3 연동)
    - 개인정보 암호화 처리 적용
- [ ] **학습 관리 모듈**:
    - 학습 요청 처리 및 상태 관리 (진행중, 완료, 실패)
    - 학습 완료 알림 발송 로직

### Phase 3: AI 엔진 개발 및 연동 (SpringBoot)
- [ ] **AI 서비스 인터페이스 구축**: API 서버와 통신할 내부 API 구축
- [ ] **음성 처리 파이프라인**:
    - 업로드된 음성 데이터 전처리
    - 음성 합성(TTS) 모델 학습/튜닝 연동
- [ ] **대화 엔진 구현**:
    - 성향 기반 템플릿 매칭 로직
    - 사용자 발화 의도 파악 및 응답 생성

### Phase 4: 프론트엔드 개발 (Flutter)
- [ ] **기본 골격 구축**: 라우팅, 상태 관리, 네트워크 레이어 설정
- [ ] **회원 서비스 구현**: 로그인, 회원가입 화면
- [ ] **전화번호부(메인) 구현**:
    - 대상 리스트 및 상세 화면
    - 대상 추가/삭제 기능
- [ ] **학습 데이터 관리 구현**:
    - 음성 녹음 또는 파일 업로드 UI
    - 성향 입력 폼
    - 학습 진행 상태 시각화 (Progress Bar 등)
- [ ] **통화 기능 구현 (핵심)**:
    - 전화 걸기/받기 인터페이스
    - 통화 연결음 및 대기 화면 (리얼타임 경험 유도)
    - 음성 입출력 처리

### Phase 5: 테스트 및 배포
- [ ] **통합 테스트**: 프론트엔드-백엔드-AI 엔진 연동 테스트
- [ ] **보안 테스트**: 개인정보 유출 및 데이터 악용 시나리오 점검
- [ ] **인프라 배포**: AWS 환경 구축 및 배포
- [ ] **출시 준비**: 스토어 등록 정보 작성 및 심사 요청

## 4. 비기능 요구사항 (Non-Functional Requirements)

1.  **보안 (Security)**
    - 모든 개인정보(이름, 이메일, 음성 데이터 등)는 DB 저장 시 암호화해야 한다.
    - 전송 구간(Client-Server)은 HTTPS를 적용하여 데이터를 보호한다.
    - 학습된 모델이 악용되지 않도록 접근 제어 및 모니터링 체계를 갖춘다.

2.  **사용자 경험 (UX)**
    - **리얼리티**: 실제 전화와 유사한 UI 구성 및 연결음, 통화 품질을 모사한다.
    - **응답성**: AI 응답 생성 시간 동안 자연스러운 대기음이나 필러(Filler) 워드를 사용하여 끊김 없는 경험을 제공한다. (비동기 처리 필수)

3.  **성능 (Performance)**
    - 다수의 사용자가 동시에 통화를 시도하더라도 안정적인 서비스를 제공해야 한다.
    - 학습 프로세스는 백그라운드에서 비동기로 처리되어야 하며, 메인 서비스 성능에 영향을 주지 않아야 한다.
