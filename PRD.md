# 프로젝트 개요
- 사랑하는 사람의 목소리와 성향을 AI에게 학습시킨다. 
- 사용자에게 이를 통해, 그 사람과 마치 전화 통화를 하는 것 같은 경험을 선사한다.
- 고인을 재현하는 AI가 아니라, 제한된 템플릿 기반 감성 음성 응답 시스템이다.

# 프로젝트 유의사항
- 학습된 음성 데이터가 악용될 여지가 있으며, 이에 대한 고려가 반드시 필요하다.
- 개인정보를 처리 및 관리해야 하니, 관련 법령을 준수해야 한다.

# 비기능 요구사항
- 개인정보 관련 데이터는 암호화가 필요하다.
- UX를 고려해서 사용자에게 최대한 실제 전화를 하는 것 같은 경험을 제공해야 한다.
  - 이를 위해 적극적으로 비동기 요소를 넣어야 한다.

# 기능 요구사항
- 회원가입 및 로그인 기능을 제공하고, 이를 통해 개개인을 식별할 수 있어야 한다.
  - 회원가입 시에는, 이메일 및 비밀번호만 가지고 수행한다. 
- 사용자는 학습시키고자 하는 대상과 관련한 데이터를 업로드 및 삭제할 수 있어야 한다.
- 사용자는 대상에게 전화를 걸 수 있고, 전화통화할 수 있어야 한다.
- 사용자는 학습의 진행사항을 실시간, 혹은 준실시간으로 확인할 수 있어야 한다.
- 사용자로부터 대상의 성향 정보를 프로필 형식으로 제공받을 수 있어야 한다.
- 학습이 완료되면 사용자에게 통지할 수 있어야 한다.
  - 사용자는 대상을 전화번호부에 추가할 수 있다.
  - 사용자는 대상을 전화번호부에서 삭제할 수 있다.
  - 사용자는 전화번호 부에 등록된 대상 리스트 및 상세를 볼 수 있다.

# 기술 스택
- 프론트엔드: Flutter 
- API 서버: SpringBoot(Tomcat) 
- AI 엔진: SpringBoot(Tomcat)
- 인프라 : AWS
- 협업도구 : Github, Git
